# Project structure

Ambiguity Compiler is intentionally organized as one web application with clearly separated local integrations and demo material.

```text
.
├── src/                 # TanStack Start web dashboard and shared app logic
│   ├── components/ac/   # Product-specific interface components
│   ├── components/ui/   # Reusable UI primitives
│   ├── hooks/           # Browser and compilation hooks
│   ├── lib/ac/          # Contracts, storage, verification, and test helpers
│   └── routes/          # File-based application routes
├── mcp-server/          # Stdio MCP server and its core tests
├── codex-plugin/        # Installable local Codex plugin and skill
├── fixtures/            # Self-contained judge/demo fixtures
├── scripts/             # Verification, fixture, and plugin validation scripts
├── public/              # Intentional browser-served assets and sanitized evidence
├── docs/                # Architecture and submission documentation
└── artifacts/           # Local generated reports; not source material
```

## Conventions

- Keep product-specific UI in `src/components/ac/`; reusable primitives remain in `src/components/ui/`.
- Keep routes thin. Put reusable behavior in `src/lib/ac/` or `src/hooks/`.
- Keep MCP-only dependencies and tests inside `mcp-server/`.
- Put runnable example projects in `fixtures/`, not under the application source tree.
- Use `scripts/` only for executable maintenance and validation utilities.
- `artifacts/test-reports/`, `.data/`, build directories, local logs, and `.env` files are generated/local-only and ignored.
- `public/verification/` contains intentionally sanitized evidence that the dashboard can display; do not place secrets or raw test reports there.

## Generated files

- `src/routeTree.gen.ts` is generated by TanStack Router. Do not edit it by hand.
- `.output/`, `.tanstack/`, `.wrangler/`, and `node_modules/` are generated runtime/build directories.
- The running development server may write `dev-server.out.log` and `dev-server.err.log`; they are ignored and should not be committed.
